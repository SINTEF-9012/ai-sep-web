Selecting an AI framework for your project

Below we present a comparison table that gives you some guidelines for selecting an AI framework that best fits your data science project. We define some selection criteria to help you decide:

  • Project size (team size): The size of the project and team.
  • Maturity level: We classify prototypes/models according to their maturity level. An experiment is the lowest maturity level. It represents a "one-shot" effort to build something to test some ideas. A project artefact represents a solution/model that is produced as a formal deliverable and used by external partners. A flagship artefact are mature solutions/models that is reused in several projects and thus requires maintenance over longer period of time. It involves a project team of at least 2 persons.
  • Popularity: How popular are the framwork with other (professionals). We base this on the study presented in the paper "Identifying the most Common Frameworks Data Science Teams Use to Structure and Coordinate their Projects" by Saltz et al.
  • Technology dependencies: Whether the framework is oriented towards a specific technology platform.
  • Tool support and available documentation: Availability of tools and documentation (e.g., user guides) that makes it easy to start using the framework.
